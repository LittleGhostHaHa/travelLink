旅行行程 - 外网访问指南

本指南提供多种方法，帮助您将本地运行的旅行行程网页分享给他人访问。请选择适合您的方法。

方法一：使用 ngrok（推荐）

1 访问 ngrok官网 下载适合您系统的版本
2 解压下载的文件，找到 ngrok 可执行文件
3 打开新的命令行窗口，运行以下命令（确保当前本地服务器已在端口8000运行）： 
ngrok http 8000
4 运行成功后，ngrok 会显示类似这样的信息： 
Forwarding                    http://1a2b3c4d.ngrok.io -> http://localhost:8000
5 将生成的外网链接（如 http://1a2b3c4d.ngrok.io）分享给他人
✅ 他人访问该链接时，会自动跳转到您本地的旅行行程网页

方法二：使用 localtunnel

1 确保已安装 Node.js（如未安装，可从 官网 下载）
2 打开命令行窗口，安装 localtunnel： 
npm install -g localtunnel
3 确保本地服务器已在端口8000运行，然后运行： 
lt --port 8000
4 运行成功后，localtunnel 会显示类似这样的信息： 
your url is: https://random-name.loca.lt
5 将生成的外网链接分享给他人

方法三：部署到 GitHub Pages

1 在 GitHub 上创建一个新的仓库
2 将您的旅行行程文件（index.html, print.html, route_map.html 等）上传到该仓库
3 在仓库的 Settings 页面中，找到 GitHub Pages 部分
4 从 Source 下拉菜单中选择 master 或 main 分支
5 点击 Save，GitHub 会自动部署您的网站
6 几分钟后，您可以通过类似这样的链接访问您的网站： 
https://your-username.github.io/your-repository-name/index.html
⚠️ 重要提示：
  • 使用ngrok或localtunnel生成的链接仅在您的本地服务器运行期间有效
  • 这些工具生成的链接可能会在每次重新启动时更改（ngrok提供付费版本可获取固定域名）
  • 分享的链接将直接暴露您本地服务器上的内容，请确保不会意外分享敏感信息
  • 如需长期稳定访问，建议使用GitHub Pages等静态网站托管服务
